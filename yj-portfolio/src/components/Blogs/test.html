Interactive Mode

  • Typically used during the development and debugging stages of a program. Allow users to see outputs in real-time and test sections of the code.
  • “node” refers to a single machine or server within a larger cluster of machines. Each node can have multiple CPUs and each CPU has multiple cores.
  • You would want to use multiple nodes when your computational task can be split into smaller tasks that can run concurrently and independently.
    • Memory requirement: if the task requires more memory (RAM) than a single node
# -n is for how many CPU cores
# -N is for node number
# -t is for time, ex) 0 days and 4 hours
interactive -n 8 -N 1 -t 0-4:00

# Requesting a gpu
interactive -G a100:1

# Assigning 
interactive --gres=gpu:a100:1 --mem=40G -p htc

# For checking if the GPU is assigned
navidia-smi

# For CUDA access checking
tf.test.is_gpu_available(cuda_only=True)

Batch Mode(sbatch)

  • Would use when you’re confident your program is working as expected and want to test it on larger datasets or perform more time-consuming calculations.
  • You would submit it as a job to a job scheduler like Slurm using the ‘sbatch’ command
    it will queue your job and run it when resources are available.
  • Allows efficient usage of computational resources in high-performance computing (HPC) environment
module load mamba/latest
source activate functa_test

sbatch myjob.sh
  • general: this comprises of all Research computing-owned CPU nodes.
    SBATCH -p general
SBATCH -t 7-00:00:00
SBATCH -c 5     # CPUs per TASK
SBATCH -n 10    # number of TASKS
SBATCH -N 10    # number of nodes to allow tasks to spread across (MIN & MAX)
SBATCH --mem=120G # Every single node in Sol has at least 500GB of memory for CPU
# If you have 4 GPUs assigned then you will have 160GB of memories no matter what[
SBATCH --gres=a100:4 # For GPU a100 has 40GB, 4 per node
# The above example will allocate 50 cores, 5 cores per task on 10 independent nodes.
  • htc: aimed at jobs that can complete within a four-hour wall time. This partition includes not only Research Compugin-owned nodes but also privately-owned nodes.
    SBATCH -p htc
  • highmen: This partition is aimed at jobs that can require an extra amount of memory that exceeds Sol’s compute node standard of 512GB.

  • Example - save this file as myjob.sh
#!/bin/bash

#SBATCH -N 1            # number of nodes
#SBATCH -c 1            # number of cores 
#SBATCH -t 0-01:00:00   # time in d-hh:mm:ss

#SBATCH -q public       # QOS
#SBATCH -o slurm.%j.out # file to save job's STDOUT (%j = JobId)
#SBATCH -e slurm.%j.err # file to save job's STDERR (%j = JobId)
#SBATCH --mail-type=ALL # Send an e-mail when a job starts, stops, or fails
